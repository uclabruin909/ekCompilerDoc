Configuration

compileConfig.json

The compileConfig.json file is used to configure the compiler prior to initializing. It contains relevant project information such as the "Company Name" , "Project Name", the date the project was started on, etc. The most imporant configuration that needs to be done is setting the root directory where the project folder will be created and setting the source file that will be used to compile each of the different UserControls and Layout files.

  • companyName: This is the name of the company that the site will be created for. The description.txt file will be written with the name provided.

  • projectName: This will control the name of the project directory created. In this example, the project directory will be called "ST_www_tompkinsfinancial_com_01".

  • projectDate: The date in which the project is started on. The description.txt file will be written with the date provided.

  • rootDir: Specify the path where you want the project directory to be initialized.

  • initDirectories: These are all the directories that will be created when the project is first initialized. By default, there will be 4 directories created:

    • Layouts- where the layouts.aspx files will be compiled into.
    • UserControls- where the UserControls files (Smarsh_Head, Smarsh_Header, Smarsh_Footer) will be compiled into.
    • xslt- the xslt files will be created inside here
    • Content- all the content blocks that will you compile will be converted into seperate HTML files and stored in this directory.

  • UserControls.files: Contains an array of objects for the different UserControl files that will be compiled. Each object contains the name, fileType, fileOrigin, and 'srcFile of that particular file. You must specify the 'srcFile' for each of the files you will be compiling. This will allow the compiler to know which file to use for the designated UserControls file.

  • Layouts.files: Contains an array of objects for the different Layout files that will be compiled. Each object contains the name, fileType, fileOrigin, and 'srcFile of that particular file. You must specify the 'srcFile' for each of the files you will be compiling. This will allow the compiler to know which file to use for the designated Layout file.

  • compileFiles: Contains a collection of files that you wish to compile that are not designated to a specific Ektron file. For instance, if there are particular files that you wish to compile the content from, add the file path to the array. When you run the compileFiles command, the compiler will go through each of these files and compile the HTML block that is wrapped in the ek-section="body" tag.

assetFile.json

Within the assetFile.json, you can create a JSON object with the asset key and it's Ektron content ID. When you specify an ek-asset="[key]", the content will be compiled with the appropriate path. This will make asset management much more maintainable and will save time with the initial configuration.

EXAMPLE:

1. Configure the assetFile.json by specifying the asset key and it's corresponding ID. In this example, we will be focusing on using the "section-about-image".

2. Once an asset and it's key has been configured in the assetFile.json, you can use the ek-asset="[key]" attribute on the desired element to automatically have the compiled source point to the appropriate id. In this example, we see which element is given this attribute.

3. Once the content is compiled, you can see that the src has been changed to the correct ID.

Commands

init

After the compiler has been configured, the init command is used to initialize the project folder. Before any compiling can be done, this directory needs to be executed. This command will create the base project folder in the rootDir path assigned in the compileConfig.json file.

EXAMPLE:

By running the init command in the terminal, the project directory will be initialized using the information that was configured within the compileConfig.json file.

remove

If you want to permanently remove the initialized project directory, simply run the remove command in the terminal.

compileFiles

Often times, not all of our files will be used as the srcFile for a UserControls or Layouts file. However, if you still want to compile a group of files, you can use the compileFiles command to do so. You specify these files within the compileFiles array property in the compileConfig.json file.

In this case, if you run the compileFiles command in the terminal, each of the files listed in the array of the "compileFiles" property will be compiled.

NOTE: The portion of the HTML that needs to be compiled still needs to be wrapped in the ek-section="body" tag.

compileAll

To compile everything at once, use the compileAll command. This will compile all UserControls and Layouts files using the srcFile assigned to each of the files in the compileConfig.json.

Compiler Tags

ek-section

The ek-section attribute allows the user to designate the areas of the HTML that will be compiled. By segmenting different parts of the HTML with an ek-section element, the compiler will know exactly which block of HTML to use to compile the Smarsh_Header, Smarsh_Footer as well as the different layout files. Remember, it will use the srcFile specified in the compileConfig.json file.

  • header: Within the srcFile configured to compile the Smarsh_Header, wrap the appropriate block of HTML that will be used for the compilation of the header. The compiler will parse this particular block of HTML, compile the different aspects of it, and then create the Smarsh_Header.ascx file inside the UserControls directory.

    EXAMPLE:

    1. In the srcFile configured to compile the Smarsh_Header file, wrap the desired block of header HTML with an element containing the ek-section="header" attribute.

    2. When you compile the Smarsh_Header.ascx file, you can see that only the block of HTML that was wrapped in the ek-section="header" tags was used during compilation. Rest of the HTML in the srcFile has been omitted.
